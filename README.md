# AI 求职助手 Agent

一个适合应届生展示的 Streamlit 小项目：支持上传或粘贴简历，输入岗位 JD、目标岗位方向和城市，调用 OpenAI 兼容格式的中转 API，生成岗位关键词、核心要求、匹配度评分、投递建议、简历优化稿、HR 打招呼话术、面试准备问题、招聘搜索关键词和真实岗位侦察结果，并导出 Word / PDF 简历。

## 项目特点

- 使用 Python + Streamlit，结构简单，容易运行和演示。
- 使用 `.env` 管理 `AI_API_KEY`、`AI_BASE_URL`、`AI_MODEL`。
- 支持页面内使用自己的 API Key 登录：可选择 OpenAI 官方或 OpenAI 兼容中转站，填写 Base URL、API Key 和模型名后直接使用。线上部署默认不提供服务器 Key，访问者必须使用自己的 Key。
- API 调用按 OpenAI 兼容的 `/chat/completions` 格式处理。
- 支持 API 超时重试、可选备用模型，降低中转 API 偶发失败对演示的影响。
- 根据城市和岗位方向生成招聘网站搜索链接，方便继续查看实时招聘信息。
- 使用浏览器自动化尝试爬取公开招聘页，提取真实岗位标题、薪资、城市、公司、技能标签和具体岗位详情页链接。
- 支持企业资质与风险评分：对岗位公司检索公开信息、投诉、欠薪、失信、经营异常、培训贷等风险信号，帮助规避可疑公司。
- 根据目标岗位给出热门能力标签，提示简历中已体现和建议补充的能力。
- 支持上传 `.txt`、`.docx`、`.pdf` 简历文件并自动识别文本。
- 根据真实岗位信息和热门能力标签手动生成可复制的简历优化稿，避免分析阶段自动消耗额外 API token。
- 简历优化稿支持在页面内预览、编辑，再把模板和简历一起交给 API 生成 Word / PDF 版本。
- Word / PDF 生成拆成独立按钮：点击生成 Word 只生成 Word 版本，点击生成 PDF 只生成 PDF 版本；生成后可继续在线编辑、刷新文件并下载。
- 内置本地使用记录：简历优化稿、手动编辑稿、Word 生成和 PDF 生成会写入 `output/history.json`，导出文件保存到 `output/history_files/`，可以在侧边栏查看历史并载入旧内容继续编辑。
- 如果上传的是 `.docx`，生成 Word 时会参考模板文字结构，并优先替换模板中文字以保留样式、图片和版面结构；模板中有 `{{AI_RESUME}}` 或 `{{简历正文}}` 时会在占位符位置插入内容。
- 内置多套简历导出模板：简洁商务、技术清爽、实施售前、应届生一页，并提供模板说明预览。
- 支持导入自定义 `.docx` / `.pdf` 简历或模板。DOCX 可预览段落、表格、图片数量；PDF 可预览页数和可识别文字。模板可识别内容会作为上下文交给 API 生成对应版本简历。
- 新增旧模板简历替换工具：默认使用 `templates/default_resume_template.docx`，也可上传自定义 DOCX 模板；系统只替换 DOCX 底层 `w:t` 文字节点，不改文本框、线条、颜色、图片和位置。
- 内置详细职业方向问卷，适合不知道投什么岗位的用户，根据专业背景、技能、沟通偏好、压力偏好、公司偏好和长期发展方向推荐岗位。
- 页面展示 Agent 执行过程和低调加载动画：读取简历、爬取岗位、分析 JD、匹配评分、记录岗位画像、等待手动生成简历。
- 输出 Markdown，方便复制到 Word、简历文档或求职记录中。
- 体现基础 Agent 思路：理解任务、提取岗位要求、分析匹配度、生成行动建议。

## 运行步骤

1. 进入项目目录：

```bash
cd ai-job-agent
```

2. 安装依赖：

```bash
pip install -r requirements.txt
```

如果要使用真实岗位侦察功能，需要本机已安装 Node.js，并能通过 Playwright 打开浏览器页面。本项目会优先尝试用浏览器自动化读取公开招聘页面；如果招聘平台要求登录、验证码或安全验证，可能只能显示部分结果。

3. 复制环境变量示例：

```bash
copy .env.example .env
```

4. 修改 `.env`：

```env
AI_API_KEY=你的中转API密钥
AI_BASE_URL=https://你的中转站地址/v1
AI_MODEL=你的模型名称
AI_FALLBACK_MODELS=备用模型1,备用模型2
AI_TIMEOUT_SECONDS=120
AI_MAX_RETRIES=2
```

5. 启动网页：

```bash
streamlit run app.py
```

浏览器打开 Streamlit 给出的本地地址后即可使用。

## API 登录方式

线上部署时，本项目只使用页面左侧“AI 接口登录”中填写的配置。访问者必须填写自己的 API Key、Base URL 和模型名后才能调用 AI。

本地开发时可以保留 `.env` 作为参考，但当前应用代码不会在用户未填写页面 API Key 时回退使用服务器 Key。

页面登录支持：

- OpenAI 官方：默认 Base URL 为 `https://api.openai.com/v1`
- OpenAI 兼容中转站：填写中转站提供的 Base URL
- 自定义模型名
- 备用模型，多个模型用英文逗号分隔

页面中填写的 API Key 只保存在当前浏览器会话，不写入历史记录，不保存到服务器文件。

## 部署到公网

如果希望别人用手机或电脑浏览器访问，并且不依赖你的电脑一直开机，建议部署到云平台。

### 方案一：Streamlit Community Cloud

适合纯 Python 演示，部署最简单。

1. 把项目上传到 GitHub。
2. 登录 Streamlit Community Cloud。
3. 选择仓库、分支和入口文件 `app.py`。
4. 部署后获得公网 URL。

注意：真实岗位侦察依赖 Node.js + Playwright，Streamlit Community Cloud 对浏览器自动化支持不如 VPS/Render 稳定。

### 方案二：Render / Railway

更适合当前项目，因为可以安装 Python、Node.js 和浏览器依赖。

启动命令：

```bash
streamlit run app.py --server.port $PORT --server.address 0.0.0.0
```

部署建议：

- Python 依赖使用 `requirements.txt`
- Node.js 用于 `browser_job_crawler.mjs`
- 如果平台需要浏览器依赖，需要额外安装 Playwright Chromium
- 不要把自己的真实 API Key 写进代码
- 不要配置服务器 `AI_API_KEY`，让用户在页面侧边栏输入自己的 Key

### 方案三：云服务器 / VPS

适合最稳定的长期演示。

1. 购买云服务器。
2. 安装 Python、Node.js、Playwright。
3. 上传项目代码。
4. 使用 `streamlit run app.py --server.port 8501 --server.address 0.0.0.0` 启动。
5. 配置 Nginx 反向代理和域名。
6. 用 `systemd` 或进程管理工具保持服务常驻。

如果只是放进简历让面试官试用，Render 或 Railway 会比自己维护服务器省事。

## 招聘链接说明

当前版本会尝试爬取公开招聘页面，并优先从前程无忧等可公开渲染的页面中提取真实岗位信息，包括岗位名、薪资、城市、公司、技能标签和具体岗位详情页链接。同时也会保留 BOSS直聘、智联招聘、前程无忧、猎聘和 Bing 的搜索链接，方便你手动复查。

你可以先点击推荐链接查看真实岗位，再把感兴趣岗位的 JD 复制回页面进行精细分析。

## 企业风险评分

真实岗位侦察结果中，每个岗位卡片都可以点击“评估企业风险”。系统会基于企业名称和岗位描述检索公开网页摘要，识别诈骗、培训贷、押金、欠薪、失信、被执行人、经营异常、严重违法、虚假招聘和投诉等风险信号，并给出 0-100 分的风险评分。

页面也提供“手动查询企业资质与风险评分”，可以输入任意企业名称进行复查。结果会同时给出国家企业信用信息公示系统、企查查、天眼查和风险关键词搜索入口，方便手动核验。

注意：企业风险评分基于公开搜索摘要和关键词规则，只作为求职风险提示，不等同于法律结论或工商结论。正式投递前仍建议手动核验营业执照、经营状态、司法风险和招聘收费情况。

## 简历上传与优化

页面支持上传 `.txt`、`.docx`、`.pdf` 简历文件。系统会先识别出简历文本，用户可以检查识别结果，再点击开始分析。开始分析只生成岗位匹配分析、岗位侦察结果和推荐链接；如果需要简历优化稿，需要在“简历优化稿”区域手动点击“生成简历优化稿（调用 AI）”。

注意：普通文字版 PDF 可以解析；扫描图片版 PDF 需要 OCR，当前版本暂不支持。

生成简历优化稿后，页面会提供三个区域：AI 完整结果、可编辑简历正文、下载前预览。用户可以先在页面里修改正文，再按需点击“生成 Word 版本（调用 AI）”或“生成 PDF 版本（调用 AI）”。Word 和 PDF 分开生成，点击 Word 只调用一次 API 生成 Word 版本正文，点击 PDF 只调用一次 API 生成 PDF 版本正文。

生成 Word / PDF 时，系统会把模板可识别文字、用户简历、真实岗位信息和热门能力标签一起交给 API，让模型按模板结构生成对应版本的简历正文。生成后可以在页面内预览和继续编辑，再刷新文件并下载。

系统会把关键使用记录写入本地历史，包括简历优化稿、手动编辑稿、Word 生成和 PDF 生成。一次生成动作只保存一条历史记录。侧边栏的“使用记录”可以查看最近记录，也可以把历史内容载入当前页面继续修改和导出。

如果上传的是 Word `.docx`，系统会把模板中可识别文字交给 API 作为格式参考。导出 Word 时，系统优先替换模板中的文字内容并保留样式、图片和版面结构；如果模板里有 `{{AI_RESUME}}` 或 `{{简历正文}}` 占位符，则会在占位符位置插入生成内容。

导出普通 Word / PDF 时，可以选择简洁商务、技术清爽、实施售前、应届生一页等模板。上传 `.docx` 时，保留格式版本会优先保留原简历自身样式；如果模板有占位符，会把内容插入到对应位置。

也可以额外导入一个 `.docx` 或 `.pdf` 作为模板。这个文件可以是已有简历，也可以是真正的空白模板。

DOCX 模板会展示段落数、表格数、图片数、节数和部分文字内容。生成 Word 版本时，会优先使用导入模板的结构。

PDF 模板会展示页数和可识别文字。生成 PDF 版本时，系统会把 PDF 模板可识别文字交给 API 作为结构参考，生成新的 PDF 简历，不再把内容追加到 PDF 模板后面。

## 旧模板简历替换工具

页面内置“旧模板简历替换工具”，用于把原始简历内容套入固定 Word 模板。

核心逻辑：

- 默认模板为 `templates/default_resume_template.docx`
- 支持上传自定义 `.docx` 模板
- 后端使用 `zipfile + xml.etree.ElementTree` 读取 DOCX 内部 `word/*.xml`
- 只提取和替换 `w:t` 文本节点
- 不使用 `python-docx` 重建文档
- 不修改模板格式、文本框、线条、颜色、图片和位置
- AI 只返回 JSON 数组：`[{"index": 1, "new_text": "..."}]`
- 每段 `new_text` 会按模板原文字长度截断，尽量保持一页
- 页面允许用户编辑替换 JSON 后再生成 Word

隐私处理：

- 页面不会显示模板中的任何原始文字
- 默认模板使用项目内脱敏副本
- 输出文件保存到 `output` 目录，并提供 `.docx` 下载

## Agent 工作流

点击开始分析后，页面会显示 Agent 执行过程：

```text
读取简历内容
生成搜索关键词和热门能力标签
爬取公开招聘信息并抽取岗位卡片
调用大模型分析 JD、岗位要求和简历匹配度
记录岗位画像并等待手动生成简历
准备分析结果和岗位链接
```

这样可以清楚展示它不是普通聊天框，而是一个会读取文件、调用工具、检索岗位、分析信息并生成投递材料的 Agent 工作流。

## 安全注意事项

- 不要把真实 API Key 写进代码或 README。
- `.env` 已加入 `.gitignore`，不要提交到 Git。
- 中转 API 由第三方服务提供，稳定性、费用、日志保留和隐私策略需要你自己确认。
- 招聘链接会跳转到第三方平台，岗位是否仍在招聘以平台实时页面为准。
- 岗位侦察只读取公开页面，不绕过登录、验证码、付费限制或平台安全验证。
- 简历内容可能包含手机号、学校、项目经历等个人信息，测试时建议先脱敏。
- 程序不会主动保存简历和 JD，但第三方 API 服务可能会记录请求内容，请谨慎粘贴敏感信息。
